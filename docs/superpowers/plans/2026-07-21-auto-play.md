# autoPlay Option Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为 `MockLivePhoto` 增加默认开启的 `autoPlay` 公共属性，并在 Expo 52 Example 中提供可见开关。

**Architecture:** `autoPlay` 只存在于 TypeScript 就绪协调层。每个资源版本创建时保存属性快照，双资源 ready 后始终通知 `onLoad`，仅快照为 `true` 时发出一次 `play` 命令；原生播放器无需修改。

**Tech Stack:** TypeScript、React Native、Bun test、Biome、Expo SDK 52。

## Global Constraints

- `autoPlay?: boolean` 默认 `true`。
- `false` 时资源 ready 后保持封面，首次点击正常播放。
- 单独更新 `autoPlay` 不影响当前资源版本；下一资源版本采用新值。
- Example 切换时使用新的组件 key 重建演示实例。
- 不修改 Swift/Kotlin 播放状态机，不增加依赖。
